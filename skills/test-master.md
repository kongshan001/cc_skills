# Test Master

## 背景需求

编写测试、制定测试策略、构建自动化框架时，需要专业指导。不同类型测试（单元、集成、E2E）有不同的最佳实践。

## 目标

提供测试策略和自动化框架支持，覆盖多种测试场景。

## 设计方案

### 测试类型
- 单元测试 (Unit Test)
- 集成测试 (Integration Test)
- 端到端测试 (E2E)

### 附加能力
- 覆盖率分析
- 性能测试
- 安全测试

## 本地部署

```bash
clawhub install test-master
```

## 效果展示

技能提供：
- 测试策略制定
- 自动化框架搭建
- 覆盖率指导
- 性能/安全测试建议
