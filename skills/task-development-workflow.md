# Task Development Workflow

> TDD-first 开发工作流

## 技能描述

TDD（测试驱动开发）优先的开发工作流，包含结构化规划、任务追踪和基于 PR 的代码审查。当需要构建软件项目、明确需求、任务拆解时触发。

## 核心功能

1. **TDD 优先工作流** - 测试驱动开发流程
2. **结构化任务规划** - 任务拆分和规划
3. **任务进度追踪** - 跟踪开发进度
4. **PR 代码审查** - 基于 PR 的代码审查流程
5. **需求clarification** - 需求确认和澄清

## 安装方式

```bash
clawhub install task-development-workflow
```

## 推荐安装评估

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 本地开发 | ⭐⭐⭐⭐⭐ | 强烈推荐，TDD 开发者必备 |
| ECS/服务器 | ⭐⭐⭐⭐ | 适合团队协作流程 |
| 个人项目 | ⭐⭐⭐⭐⭐ | 提升代码质量 |

### 适用人群

- 追求代码质量的开发者
- 需要结构化开发流程的团队
- TDD 实践者
