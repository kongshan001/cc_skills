# Godot Dev Guide - 完整游戏开发指南

## 技能描述
Godot 4.x 完整开发指南技能，涵盖GDScript模式、文件格式（.gd/.tscn/.tres）、场景架构、物理/UI/音效、性能优化、多平台导出和测试。专为AI辅助开发设计，自动触发于Godot相关开发任务。

## 功能列表
- **文件格式处理**: GDScript代码、场景序列化、资源序列化
- **类型系统**: 变量类型定义、函数签名规范、类型安全
- **架构模式**: 组合优先于继承、信号通信、资源配置、自动加载
- **项目结构管理**: 场景组织、脚本管理、资源目录、自动加载目录
- **常用节点**: CharacterBody2D/3D、RigidBody2D/3D、Area2D/3D、Sprite2D/MeshInstance3D
- **变量导出**: @export注解、@export_range、@export_file等
- **信号系统**: 信号定义、信号发射、事件处理
- **节点引用**: @onready变量、场景查找、组件管理
- **物理系统**: 碰撞检测、物理模拟、刚体动力学
- **UI与输入**: UI系统、用户输入处理、设备支持

## 安装方式
```bash
# 直接使用，无需额外安装
# 确保环境包含Godot引擎（4.6+版本推荐）

# 基本项目结构
res://
├── project.godot
├── scenes/           # .tscn 场景
│   ├── player/
│   ├── enemies/  
│   └── ui/
├── scripts/          # .gd 脚本
├── assets/           # 资源文件
├── autoload/         # 单例脚本
├── resources/        # .tres 资源
└── test/             # 测试
```

## 关键特性
- **区分文件格式**: .gd（完整GDScript）vs .tres/.tscn（序列化格式）
- **类型安全**: 强类型变量定义，避免运行时错误
- **架构设计**: 组合优于继承，信号用于解耦通信
- **自动触发**: 自动识别Godot相关开发需求

## 推荐安装评估

### 本地部署
**适用场景**:
- 个人开发者、小型游戏项目
- 需要快速原型开发和迭代
- 学习Godot引擎的开发者
- 离线工作环境

**优势**:
- 完全本地化，无网络依赖
- 开发环境配置简单
- 快速启动，即时反馈
- 学习成本低

**ECS部署**
**适用场景**:
- 大型游戏项目团队
- 多人协作开发
- 需要统一规范和代码审查
- 企业级项目管理

**优势**:
- 团队协作功能完善
- 统一的开发标准和规范
- 集成CI/CD和版本控制
- 项目管理和进度跟踪
- 资源统一管理

**选择建议**: 个人学习和小项目推荐本地部署，团队项目和企业开发选择ECS部署。
