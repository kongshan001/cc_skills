# Ultrathink - Claude Code 深度思考技能

## 1. 背景需求

在处理复杂编程问题时，Claude Code 通常采用快速推理模式。但在面对以下场景时，需要更深入的思考能力：

- 架构设计决策
- 复杂的 bug 调试
- 性能优化方案
- 多模块系统设计
- 算法设计与优化

## 2. 目标

Ultrathink 是一个专注于深度思考的 Claude Code 插件，通过系统性的推理过程，帮助解决需要「深思熟虑」的复杂编程问题。

## 3. 设计方案

### 核心机制

Ultrathink 通过以下方式增强 Claude 的推理能力：

1. **递归深度思考**
   - 迭代式问题分解
   - 多角度分析
   - 假设验证循环

2. **思维链外显**
   - 完整记录推理过程
   - 便于审查和调整
   - 最终总结关键决策点

3. **多方案对比**
   - 生成多个可行方案
   - 权衡利弊分析
   - 推荐最优解

### 功能列表

| 功能 | 说明 |
|------|------|
| 问题分解 | 将复杂问题拆分为可管理的子问题 |
| 深度推理 | 多轮迭代思考，探索解决方案 |
| 方案对比 | 生成并比较多个解决方案 |
| 决策记录 | 记录推理过程中的关键决策 |

## 4. 本地部署

### 安装方式

```bash
# 通过 /plugin 命令安装
/plugin install ultrathink

# 或在 Claude Code 中直接使用
/ultrathink <your complex question>
```

### 环境要求

- Claude Code 已安装
- 无需额外依赖

## 6. 优缺点分析

### 优点

| 优势 | 说明 |
|------|------|
| 深度推理 | 适合复杂问题的系统分析 |
| 方案对比 | 多角度评估，避免决策偏差 |
| 透明推理 | 推理过程清晰可见 |
| 可迭代 | 支持多轮深入思考 |

### 缺点

| 劣势 | 说明 |
|------|------|
| 响应时间 | 深度思考需要更长处理时间 |
| Token 消耗 | 详细的推理过程消耗较多 token |
| 不适合简单问题 | 对简单问题有过度使用之嫌 |

## 7. 平替对比

| 工具 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| Ultrathink | 深度思考增强 | Claude 原生集成、推理透明 | 响应较慢 |
| 思维链提示 | 通用推理增强 | 无需安装 | 需手动触发 |
| o1 模型 | OpenAI 推理模型 | 推理能力强 | 需切换模型 |

## 8. 落地过程

### 使用场景

1. **架构设计**
   ```bash
   /ultrathink 设计一个微服务架构，需要支持...
   ```

2. **复杂 bug 调试**
   ```bash
   /ultrathink 这个间歇性出现的并发 bug...
   ```

3. **性能优化**
   ```bash
   /ultrathink 如何优化这个查询从 5s 降到 100ms...
   ```

### 最佳实践

- **适用**：复杂、多因素、需要权衡的问题
- **不适用**：简单 CRUD、常规代码编写
- **建议**：配合普通模式交替使用
