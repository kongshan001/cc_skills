# 身份猜猜猜 (identity-guess-game)

## 背景需求

多人互动推理游戏是社交场景中受欢迎的娱乐形式，但传统线下游戏需要主持人、道具准备。AI 助手可以充当游戏主持，动态分配身份、引导游戏流程。

## 目标

实现一个无需主持人、自动裁判的多人身份推理游戏，每位玩家获得秘密身份，通过线索互相试探，最终猜测他人身份。

## 设计方案

- **核心机制**: 每轮玩家获得一个秘密身份，通过3轮线索描述暗示自己身份，同时试探他人
- **触发方式**: 关键词触发（身份猜猜猜、猜身份、猜猜我是谁、identity guess、猜猜猜）
- **游戏流程**: 
  1. 系统为每位玩家分配随机身份
  2. 每轮玩家描述自己的身份线索
  3. 玩家互相猜测身份
  4. 最终正确率最高者获胜

## 本地部署

```bash
clawhub install identity-guess-game
```

## 效果展示

- 多人互动推理游戏
- 自动身份分配与游戏主持
- 3轮线索描述与猜测环节

## 优缺点分析

✅ 优点:
- 无需主持人，AI 自动掌控游戏节奏
- 适合社交场景，互动性强

❌ 缺点:
- 需要多人参与，单人无法启动
- 依赖文字交流，实时性要求高

## 平替对比

| 特性 | identity-guess-game | 传统桌游 |
|------|---------------------|----------|
| 主持人 | AI自动 | 需专人 |
| 准备时间 | 即开即用 | 需准备道具 |
| 人数要求 | 灵活 | 固定 |

## 落地过程

1. 触发关键词启动游戏
2. 系统分配身份给每位玩家
3. 按轮次进行线索描述与猜测
4. 游戏结束公布结果
