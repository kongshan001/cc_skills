## 🔧 核心规则

### 1. 首先检查记忆
```
读取 ~/code/memory.md 中的用户偏好 (如果存在)
```

### 2. 用户控制执行
- 此技能提供指导，非自主执行
- 用户决定何时进入下一步
- 子代理委托需要用户明确请求

### 3. 规划后再编码
- 将请求分解为可测试的步骤
- 每步独立可验证
- 参考 planning.md 中的模式

### 4. 验证一切

| 操作后 | 建议 |
|--------|------|
| 每个函数 | 建议运行测试 |
| UI 更改 | 建议截图 |
| 交付前 | 建议完整测试套件 |

### 5. 按请求存储偏好

| 用户说 | 操作 |
|--------|------|
| "记住我偏好 X" | 添加到 memory.md |
| "永远不要再做 Y" | 添加到 memory.md Never 部分 |

- ❌ 交付未测试的代码 → 始终先验证

- ❌ 忽略偏好 → 首先检查 memory.md

## 🔐 安全与隐私

### 本地存储的数据
- 仅用户明确要求保存的偏好
- 存储在 `~/code/memory.md`

### 离开机器的数据
- 无 - 此技能不进行网络请求

### 此技能不会
- 自动执行代码
- 访问网络或外部服务
- 访问 ~/code/ 和用户项目之外的文件
- 在用户不知情的情况下采取自主行动
- 未经用户明确请求委托子代理
