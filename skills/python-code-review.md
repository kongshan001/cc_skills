# Python Code Review - Python代码审查技能

## 技能描述
Python代码审查专家技能，专注于类型安全、异步模式、错误处理和常见错误检测。用于审查.py文件、检查类型提示、async/await使用情况和异常处理，提供专业的代码质量评估和改进建议。

## 功能列表
- **PEP8风格检查**: 缩进、行长度、空白、命名规范
- **类型安全**: 类型提示检查、Any使用、联合类型语法
- **异步模式**: 阻止调用检查、await使用验证
- **错误处理**: 异常类型检查、上下文保留、日志记录
- **常见错误**: 可变默认参数、print语句使用、导入问题
- **代码质量**: 代码可读性、最佳实践、性能考虑

## 审查清单
### PEP8风格
- [ ] 4空格缩进（无制表符）
- [ ] 行长度≤79字符（文档字符串/注释≤72字符）
- [ ] 顶级定义间隔两个空行，类内间隔一个空行
- [ ] 导入分组：stdlib → third-party → local（组间空行）
- [ ] 括号内无空白，冒号和逗号前无空白
- [ ] 命名：函数/变量使用snake_case，类使用CamelCase，常量使用UPPER_CASE

### 类型安全
- [ ] 函数参数和返回类型全部有类型提示
- [ ] 不使用Any（必要时添加说明注释）
- [ ] 正确使用T | None语法（Python 3.10+）

### 异步模式
- [ ] 异步函数中无阻塞调用（time.sleep、requests）
- [ ] 所有coroutine正确使用await

### 错误处理
- [ ] 无裸露except:子句
- [ ] 具体异常类型和上下文
- [ ] raise ... from保留堆栈跟踪

## 关键规则
### 有效模式（不应标记）
- 类型注释vs类型断言：注释声明类型但非运行时断言
- 使用Any与无类型库交互：外部库缺少类型存根时必需
- 空的__init__.py文件：包结构有效，无需代码
- noqa注释：当linter规则不适用于特定情况时有效

### 上下文敏感规则
| 问题 | 仅在以下情况标记 |
|------|------------------|
| 通用异常处理 | 有具体且有意义的异常类型可用 |
| 未使用变量 | 变量无_前缀且不在f-string、日志或调试中使用 |

## 审查问题
1. 代码是否遵循PEP8格式规范？
2. 导入是否正确分组（stdlib → third-party → local）？
3. 名称是否符合约定（snake_case、CamelCase、UPPER_CASE）？
4. 函数签名是否完全类型化？
5. 异步函数是否真正非阻塞？
6. 异常是否包含有意义的信息？
7. 是否有可变默认参数？
