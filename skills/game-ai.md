# Game AI Systems

## 1. 背景需求

游戏 AI 是游戏开发的核心组成部分，开发者经常面临：
- 不知道用行为树还是状态机
- NPC 行为逻辑混乱难以维护
- 寻路算法选择困难
- 决策系统实现复杂

需要一个系统的游戏 AI 开发指南。

## 2. 目标

提供完整的游戏 AI 开发指南，涵盖：
- 行为树（Behavior Tree）设计
- 状态机（State Machine）实现
- 寻路算法（A*、NavMesh 等）
- 决策系统（Utility AI、Goal-Oriented）
- 群组 AI 和 flocking
- 动画状态融合

## 3. 设计方案

**知识模块**：
| 模块 | 内容 |
|------|------|
| 行为树 | 节点类型、组合器、装饰器 |
| 状态机 | 状态转换、层次状态机 |
| 寻路 | A*, NavMesh, Dynamic Obstacles |
| 决策 | Utility-based, Goal stacking |
| 群组 | Flocking, Steering behaviors |

**引擎适配**：指南性质，不限定引擎，但示例可能涉及 Unity/Godot。

## 4. 本地部署

```bash
clawhub install game-ai
```

**依赖**：无特定依赖（纯知识型技能）

## 5. 效果展示

触发示例：
- "帮我设计一个巡逻 NPC 的行为树"
- "解释一下 Utility AI 的工作原理"
- "游戏中敌人追击玩家用什么算法好"

执行效果：AI 提供设计建议、代码示例、实现指南。

## 6. 优缺点分析

| 优点 | 缺点 |
|------|------|
| 系统性知识覆盖 | 无具体代码实现 |
| 适合学习理解 | 需结合引擎实际开发 |
| 决策选型有指导 | 复杂场景需要更深度方案 |
| 跨引擎适用 | 没有可视化工具 |

## 7. 平替对比

| 方案 | 特点 | 适用场景 |
|------|------|----------|
| Game AI（当前） | 知识指南，设计思路 | 学习和选型 |
| 游戏开发框架（自研） | 具体实现 | 生产项目 |
| Unity Behavioral Toolkit | 商业工具 | Unity 项目 |
