# 旅行游戏副本生成器 (travel-city-game)

## 背景需求

用户想要旅行但不知道去哪好玩，传统旅行攻略信息分散且缺乏互动性。结合游戏化叙事和真实预订链接可以提升旅行规划体验。

## 目标

根据用户输入的城市名，生成游戏化3节点叙事行程 H5 页面，包含 AI 故事文案和飞猪预订链接。

## 设计方案

- **输入**: 城市名（如"杭州"、"京都"）
- **处理流程**:
  1. 调用 flyai 搜索城市景点、体验、美食
  2. 生成 3 节点叙事行程
  3. 每个节点包含 AI 故事文案 + 飞猪预订链接
- **触发方式**: /travel-city-game 城市名 或关键词触发

## 本地部署

```bash
clawhub install travel-city-game
```

## 效果展示

- 游戏化叙事行程，沉浸感强
- 真实预订链接，可直接下单
- H5 页面分享便捷

## 优缺点分析

✅ 优点:
- 旅行+游戏化创新结合
- 真实预订链路完整
- 故事化呈现有趣

❌ 缺点:
- 依赖飞猪 API，数据有限
- 仅支持有飞猪产品的城市

## 平替对比

| 特性 | travel-city-game | 普通旅行攻略 |
|------|------------------|--------------|
| 呈现形式 | 游戏化叙事 | 列表 |
| 预订能力 | 直连 | 需跳转 |
| 互动性 | 高 | 低 |

## 落地过程

1. 输入 /travel-city-game 城市名
2. 等待生成 3 节点行程
3. 点击预订链接完成预订
