# Docker Manager

## 1. 背景需求

容器化环境中，运维人员需要频繁执行容器管理操作：
- 查看运行状态和资源使用
- 启动、停止、重启容器
- 查看日志排查问题
- 批量管理多个容器

命令行操作繁琐且容易出错，需要一个更高效的助手。

## 2. 目标

提供中文 Docker 容器管理能力：
- 容器列表查看（状态、资源、端口映射）
- 启动、停止、删除容器
- 资源监控（CPU、内存、网络）
- 日志查看和搜索
- 批量操作支持

## 3. 设计方案

**核心能力**：
- 容器状态概览（running/exited/paused）
- 资源使用监控（top 类似功能）
- 日志实时查看和历史检索
- 单容器和批量操作
- 容器详情信息展示

**交互特点**：中文界面，适合中文用户快速操作。

## 4. 本地部署

```bash
clawhub install docker-manager
```

**依赖**：
- Docker CLI
- 适当权限（Docker socket 访问）

**验证**：
```bash
docker ps
```

## 5. 效果展示

触发示例：
- "查看所有容器状态"
- "停止所有运行的容器"
- "查看 web 容器最近 100 行日志"
- "哪个容器占用内存最多"

执行效果：AI 生成并执行命令，返回格式化结果。

## 6. 优缺点分析

| 优点 | 缺点 |
|------|------|
| 中文界面，易用 | 功能相对基础 |
| 操作覆盖常见场景 | 无图形化界面 |
| 结合 Docker 命令直接执行 | 批量操作需要明确定义 |

## 7. 平替对比

| 方案 | 特点 | 适用场景 |
|------|------|----------|
| Docker Manager（当前） | 中文，专注运行时管理 | 运维监控 |
| Docker Essentials | 英文，基础命令速查 | 通用操作 |
| Docker Helper | 功能更全面 | 深度使用 |

## 8. 落地过程

**推荐安装评估**：
- **本地开发机** ⭐⭐⭐ — 辅助查看容器状态
- **ECS 云服务器** ⭐⭐⭐⭐⭐ — 运维必备

**使用场景**：
1. 日常运维巡检
2. 问题排查（查看日志）
3. 资源监控
