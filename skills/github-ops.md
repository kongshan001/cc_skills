## 背景需求

开发者需要频繁进行仓库创建、代码推送、Release 发布等操作。这些操作通常需要在浏览器和命令行之间切换，效率较低。

## 目标

提供全自动化的 GitHub 操作技能，无需用户干预即可完成仓库创建、代码推送和 Release 管理。

## 功能列表

1. **仓库创建** - 自动创建新仓库
2. **代码推送** - 自动推送代码到远程仓库
3. **Release 管理** - 创建和管理项目版本发布
4. **全自动流程** - 减少手动干预

## 安装方式

```bash
clawhub install github-ops
```

## 推荐安装评估

- **本地开发**: ⭐⭐⭐⭐⭐ 非常适合
- **ECS 服务器**: ⭐⭐⭐⭐⭐ 非常适合，自动化部署利器

## 优缺点分析

### 优点
- 全自动化执行
- 减少手动操作
- 适合 CI/CD 集成

### 缺点
- 功能相对单一
- 需要配置 GitHub Token
