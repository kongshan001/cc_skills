## 📝 文档规范

每个 Skill 文档包含：

1. **Skill 背景需求** - 为什么需要这个 Skill
2. **目标** - Skill 要解决什么问题
3. **设计方案** - Skill 的技术架构
4. **本地部署** - Win/Mac/Linux 部署指南
5. **效果展示** - 实际使用效果
6. **优缺点分析** - Skill 的优势与不足
7. **平替对比** - 类似 Skill 对比
8. **落地过程** - 实践验证的完整记录

## 🔧 如何贡献

1. 在根目录下创建 Skill 文件夹
2. 按照上述规范编写 `README.md`
3. 在项目根目录 `README.md` 中添加链接
4. 提交 PR
